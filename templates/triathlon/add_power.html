  • A variety of exercise devices produce file records with the .tcx extension. These files contain data for a workout: time, heart rate, cadence, lap time, etc as well as GPS coordinates. These files are unprotected: in-fact they need to be; multiple fitness monitoring platforms rely on users being able to upload their own files. The formatting of these files follow XML guidelines, files are also human readable in a text editor, as an added bonus. This combination of factors will allow scripting to automatically spoof results. The first example is added functionality while the second is boarder line malicious.

    .tcx files have the following macro format:

    
<Header>
<DataPoints>
<Footer>

    The header and footer contain information such as the date, device used, file specifications, average values for parameters. The information here is essentially housekeeping, fairly un-interesting and will be completely left alone.

    The individual data points contain the information that we wish to change. An example data point:

    <Trackpoint>
<Time>2015-01-10T17:31:19.000Z</Time>
<DistanceMeters>723.1699829101562</DistanceMeters>
<HeartRateBpm>
<Value>101</Value>
</HeartRateBpm>
<Cadence>88</Cadence>
<Extensions>
<TPX xmlns="http://www.garmin.com/xmlschemas/ActivityExtension/v2">
<Speed>7.103000164031982</Speed>
</TPX>
</Extensions>
</Trackpoint>

    The watch pings every few seconds, and creates a block of data with the same format. Distance is in meters and speed is m/s.

    Add Power and remove GPX points:

    The first exercise involves editing data generated on my bike trainer. I would like to add power to my data so that it’s viewable on workout tracking sites. Sometimes I forget to turn off GPS when I’m in my living room, the resulting map is meaningless, so I would like to remove all maps from resulting data.

    Power guesstimate:

    Kurt Kinetic has an estimated power to speed formula which will be used to calculate power with a given speed. The resulting power figures are by no means correct (an investigation of potential errors), this is done mainly so I can track my progress. Results will be consistent but not accurate; I'll be able to see if I'm actually getting faster this season. This is simply generating a better metric than speed.
